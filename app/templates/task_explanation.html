Task Explanation

This task is based on Software Engineering Lecture 4 (SE_lec4.pdf) and implements best practices in software development.

1️⃣ Naming Conventions
  • camelCase for variables and methods: 
    def createTables():
    """
    Create database tables if they do not exist and add a default admin user.
    """
  • PascalCase for class names: 
    class User(db.Model):
  • UPPER_CASE for constants: 
    SECRET_KEY = 'my_secret_key'
2️⃣ Documentation Style
  • The app follows PEP 257 docstring format. For example: 
    def createTables():
    """
    Create database tables if they do not exist and add a default admin user.
    """
3️⃣ Error Handling
  • Proper error management using try-except blocks. For example: 
    try:
    user = User.query.filter_by(username=username, password=password).first()
    if user:
        session['user_id'] = user.id
except Exception as error:
    flash(f'Error: {error}', 'danger')
4️⃣ Version Control
  • The project uses Git for version control: 
    git init
git add .
git commit -m "Initial commit"
git push origin main
  • Recommended use of GitFlow branching strategy: 
    git checkout -b develop
git checkout -b feature-login
